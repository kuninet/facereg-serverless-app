# AI Agent Rules

このプロジェクトにおいてAIアシスタント（エージェント）が遵守すべきルールや記憶事項です。

## ドキュメント・作図に関するルール
* アーキテクチャ図やフロー図などの図解を作成する際は、なるべく **Mermaid形式** を使用すること。

## エージェントの運用・対応ルール
* エージェントチームとしてタスクを実施するなど、複数のモデルを利用可能なユースケースでは、可能な限り **Gemini 1.5 Flash等** の安価なモデルを優先して利用し、コストを抑えること。

## 🚨 開発ワークフロー（全プロジェクト共通・絶対遵守）
今後発生するすべての作業（新規機能実装・バグ修正・ドキュメント更新など）において、以下のフローを例外なく守ること。
**※人間のユーザーがこの手順の指示を忘れた場合、作業を開始する前に必ず人間へ手順の確認・問い合わせを行うこと。**

1. **Issue作成**: 作業内容に応じた Issue を GitHub 上に起票する。
2. **作業ブランチ作成**: メインブランチから、Issueに基づく個別の作業ブランチ（例: `feature/issue-XX`, `fix/issue-XX`）を作成する。
3. **作業の実施とコード品質の担保**:
   - 該当ブランチ内でのみコード修正を実施する。
   - **必ず `husky` と `lint-staged` 等を利用した Pre-commit フックを導入**し、コミット時に **ESLint と Prettier（または同等のLinter/Formatter）が自動で実行・整形(Fix)** される仕組みを構築すること。
   - 今後他のプロジェクトを立ち上げる際も、この Linter/Formatter 自動化フックの導入を初期セットアップの必須要件とする。
4. **Pull Request (PR) の作成**: 作業完了後、メインブランチへ向けて PR を作成する。この際、PRの本文(`body`)には必ず **`Closes #紐づくIssue番号`** を記載し、マージと同時にIssueが自動クローズされる仕組みを徹底すること。
5. **マージは人間が行う**: PR のマージ処理は AI アシスタント側では行わず、**必ず人間のレビューと承認・マージを待つ**こと。

## 🧪 テスト戦略と品質保証ルールの徹底 (全プロジェクト共通)
1. **可能な限りの事前テストの義務**: 実装中は、ユニットテストやローカルエミュレータ（`sam local` 等）を最大限活用し、Macローカルでの検証工程を必ず挟むこと。
2. **Playwright デュアル運用（テストランナー + CLI）**:
   - **CI/CDリグレッションテスト**: `@playwright/test` を使用する。`*.spec.ts` ファイルに `test()` / `expect()` を用いて記述し、`npx playwright test` で実行する。並列実行・リトライ・HTMLレポート・マルチブラウザ対応はこちらで担保する。
   - **AIエージェントによる動的テスト**: `@playwright/cli` を使用する。AIコーディングエージェント（Cursor, Claude Code 等）がブラウザを直接操作して画面確認・検証を行う際に利用する。テストランナー (`@playwright/test`) とは**別パッケージ・別用途**であり、混同しないこと。
   - 両方を devDependencies にインストールし、併用すること。
3. **再現性の確保**: 作成したテストケースは `docs/` 等にドキュメントとして残し、テストスクリプト自体もリポジトリにコミットし、誰でもいつでも再実行できるようにすること。

## 🤖 モデルの使い分けポリシー (全プロジェクト共通)

AIと人間の協調作業時のコスト最適化および効率化を目的として、AIエージェントを使用する際のモデル選択および役割分担に関する以下のルールを定める。

1. **修正プランの検討・高度な推論 (Planning Phase)**
   - **推奨モデル**: Gemini EXP (e.g., Gemini 3.1 Pro 等の最新・高推論モデル)
   - **役割**: 要件定義、アーキテクチャ設計、複雑なバグの調査、実装プラン (`implementation_plan.md` 等) の策定を行う。

2. **コードの修正・テスト・デプロイ作業 (Execution Phase)**
   - **推奨モデル**: Gemini 1.5 Flash / Gemini 1.5 Pro 等の高速かつ安価なモデル
   - **役割**: 策定されたプランに基づく実際のソースコード修正、CLIコマンドの実行、テストの実行、ならびにAWS等へのデプロイ作業を担う。

### ワークフローにおける実践
- 人間が新しい実装や複雑な要件を依頼する際は、より高度な推論力を持つ上位モデルを選択してプラン作成を指示する。
- プランが完成・合意できたら、人間はUI上から安価なモデル（Gemini 1.5系など）へ切り替え、その後の実際のコーディングとデプロイ作業を続行させる。
- AIエージェント自身も、単純作業やデプロイ実行プロセスに入ることが明確な場合は、ユーザーに対して「モデルをGemini 1.5 Flash等に切り替えてコストを抑えて進めても良いか」を提案すること。
