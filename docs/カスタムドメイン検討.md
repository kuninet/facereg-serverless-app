# カスタムドメインの設定について（2つの入口構成）

すでにお持ちのドメイン `example.com` を使用して、**「受付用」と「管理用」の2つの入口（サブドメイン）**を構成することは**可能**です。AWSのサーバーレス構成においては、複数のサブドメインを利用しても**維持費や証明書費用に関する追加コストは無料**で実現できます。

* **入口1（受付用）**: `https://facereg.example.com`
* **入口2（管理用）**: `https://admin.facereg.example.com`

## 概要と仕組み

1. **AWS Certificate Manager (ACM)**
   * AWSでSSL/TLS証明書（HTTPSにするための証明書）を**無料**で発行できます。
   * これにより、`https://reception.example.com` でアクセスできるようになります。
2. **Amazon Route 53 (オプション)**
   * AWSのDNSサービスです。もし現在のドメイン管理会社を利用したままにするなら、AWSのDNS料金はかかりません。
3. **Amazon CloudFront / API Gateway**
   * ACMで発行した証明書を利用し、リクエストされたサブドメイン（`facereg`か`admin.facereg`か）に応じて、単一のCloudFrontからそれぞれの画面機能を提供するルーティング設定を行います。これも追加費用なしで設定可能です。

## 実現するための手順

現在 `example.com` を管理しているドメイン管理サービス（お名前.com、ムームードメイン、Xserverなど）のDNS設定画面で、以下のレコードを追加するだけで連携が完了します。

1. **証明書発行の検証レコード作成**:
   * AWSでSSL証明書を発行する際、「本当にそのドメインの所有者か？」を確認するためのCNAMEレコード情報が提供されます。
   * これを現在のドメイン管理画面に1つ登録します。
2. **システムへのルーティング（アクセス先の指定）**:
   * システム構築完了後、AWS CloudFrontが生成する固有のURL（例: `d1234abcd.cloudfront.net`）が払い出されます。
   * ドメイン管理画面で、`facereg.example.com` および `admin.facereg.example.com` の2つの CNAME レコードを作成し、アクセス先を上記のCloudFrontのURLに設定します。

## コスト一覧

| 項目                       | 費用（月額） | 備考                                            |
| :------------------------- | :----------- | :---------------------------------------------- |
| サブドメインの管理費       | 0円          | 現在のドメイン管理会社の料金に含まれます        |
| SSL/TLS証明書代 (ACM)      | 0円          | AWSのACMを利用すれば、自動更新でずっと無料です  |
| CloudFront等への紐付け設定 | 0円          | 機能利用自体に追加費用はかかりません            |
| **合計追加コスト**         | **0円**      | 独自ドメインに関するAWS側での課金は発生しません |

*※仮にDNS管理そのものを現在の会社からAWS（Amazon Route 53）に移行する場合は、月額0.5ドル（約75円）のホストゾーン維持費がかかりますが、現状のDNSサーバーをそのまま使うのであれば一切無料です。*

## 結論

かっこいいカスタムドメイン（例： `https://reception.example.com` ）を導入することは、**現在のAWS利用料の範囲（つまり0円）で簡単に実現可能**ですので、ぜひ構成に組み込むことをお勧めします！
