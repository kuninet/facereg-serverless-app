# UI設計（ワイヤーフレーム設計）

本ドキュメントでは、顔写真登録アプリの2つの入口（受付用・管理用）における画面構成を定義します。

## 1. 2つの入口（ドメイン設計）
システムの用途を明確に分離し、セキュリティと利便性を高めるため、以下の2つのサブドメイン（入口）を構成します。

* **① 登録受付アプリ（一般ユーザー・来客用）**
  * 想定URL: `https://facereg.example.com`
  * 認証: なし（誰でもアクセス可能）
  * 特徴: キオスク端末（iPad等）での利用を想定し、シンプルで入力しやすいUI。

* **② 管理画面アプリ（社内管理者用）**
  * 想定URL: `https://admin.facereg.example.com`
  * 認証: あり（簡易ログイン画面）
  * 特徴: PCからの閲覧・操作を前提とした一覧表示・データ出力機能。

---

## 2. 画面定義

### 2.1. 登録受付画面 (`facereg.example.com`)

キオスクモードでの運用を前提に、1画面で完結するシンプルな構成にします。

#### 画面要素 (上から順に配置)
1. **ヘッダー**
   * 「ご来場ありがとうございます。情報を入力し、顔写真を撮影してください。」などの案内文。
2. **プレビューエリア（カメラ/写真）**
   * スマホ/タブレットの場合: 自動でフロントまたはリアカメラ（選択可）が起動し、プレビューを大きく表示。
   * PCの場合: 「ファイルを選択してアップロード」ボタンやドラッグ＆ドロップエリアを表示（カメラ起動UIとフォールバック）。
   * シャッターボタン（またはファイル選択ボタン）。
   * 撮影後: 撮影/選択した画像を表示（再撮影ボタン配置）。
3. **入力フォーム**
   * **日付**: 自動入力（本日日付を初期値として設定・変更不可でも可）。
   * **会社名**: テキスト入力（必須）。
   * **お名前**: テキスト入力（必須）。
   * **ご用件**: ドロップダウンリスト（例：「商談」「面接」「配達」「その他」等） ＋ 詳細テキストボックス（任意）。
4. **アクションボタン**
   * 大きな **[登録する]** ボタン。
5. **完了・リセット動作（キオスクモード）**
   * 「登録する」ボタン押下後、「登録が完了しました」という完了モーダルまたは完了画面を数秒間表示。
   * その後、自動的にフォームと写真をクリアし、初期の入力画面に戻る。

---

### 2.2. 管理画面 (`admin.facereg.example.com`)

#### 画面1: ログイン画面
1. **フォーム要素**
   * ユーザーID入力欄
   * パスワード入力欄
2. **アクション**
   * **[ログイン]** ボタン

#### 画面2: 顔写真登録一覧画面（ダッシュボード）
管理者が普段操作するメイン画面です。PCでの閲覧を想定し、情報をテーブル形式でリスト化します。

1. **ヘッダー/ナビゲーション**
   * 「顔写真登録管理」タイトル
   * ログアウトボタン
2. **検索・絞り込みエリア (画面上部)**
   * **日付指定**: Start～Endの日付ピッカー
   * **キーワード検索**: 会社名、名前、用件によるフリーテキスト検索
   * **[検索]** ボタン / **[クリア]** ボタン
3. **アクションエリア (テーブル上部)**
   * **[チェックした項目を削除]** ボタン
   * **[チェックした項目のZIP一括ダウンロード]** ボタン
     * ※ダウンロードされるZIPには、選択された顔写真画像（圧縮済みjpg/png）と、それに紐づく入力データ（CSVファイル）が含まれます。
   * **[バックアップからのリストア]** ボタン（※別画面やモーダルでZIPをアップロードさせる想定）
4. **データ一覧テーブル**
   * 列項目:
     * `[ ]` (一括操作用のチェックボックス)
     * `ID` または `No.`
     * `登録日時`
     * `顔写真` (サムネイル画像をクリックで拡大表示)
     * `会社名`
     * `お名前`
     * `ご用件`
     * `操作` (行ごとの「削除」ボタン、個別ダウンロードボタン)
   * ページネーション（例：1ページ50件表示など）
