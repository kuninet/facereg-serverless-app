# Faces Registration Serverless App (顔写真登録アプリ)

AWSのサーバーレスアーキテクチャを活用し、継続課金を極力残さない方針で運用する受付用の顔写真登録アプリケーションです。

## プロジェクト概要
受付に設置したキオスク端末（iPad等を想定）から、来訪者の顔写真と受付情報を登録し、管理者がクラウド上で一覧表示・ダウンロードできるシステムです。継続課金を避けるため、常時起動サーバーを置かず、保存データは短期保持、ログも短期保持、不要時はスタックごと削除する前提で構成します。

## 構成
* **フロントエンド**: React/Vite 等（S3 + CloudFrontホスティング）
* **バックエンド**: AWS SAM (API Gateway + Lambda + DynamoDB)
* **ストレージ**: Amazon S3 (署名付きURLによる直接アップロードを利用)

## コスト運用方針
* X-Ray トレースは無効化し、CloudWatch Logs の保持期間は 7 日に制限します。
* 顔写真と受付データは 7 日で自動削除し、長期保管による放置課金を避けます。
* 利用しない期間は `./scripts/destroy-stack.sh` でスタックごと削除し、フロント配信や保存領域の残存コストを止めます。

## 環境ごとの変数の管理について
当リポジトリはパブリック公開用のため、インフラに関する固有のリソース名やカスタムドメイン名は、コード・ドキュメント内でダミー値（例: `example.com`）にマスキングされています。
実際の運用環境にデプロイする際は、以下の方法で環境固有の値を管理してください。

1. **フロントエンド環境変数**
   `.env.example` をコピーして `.env.local` を作成し、割り当てられたAPIエンドポイント等の値を設定してください。
2. **バックエンド (AWS SAM) 管理**
   `.env.deploy.example` をコピーして `.env.deploy` を作成し、`ADMIN_AUTH_CREDENTIALS` などのデプロイ用機密情報をローカル管理してください。`samconfig.toml` と `.env.deploy` は Git 管理から除外されています。
